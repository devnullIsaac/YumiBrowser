# Code of Conduct

Yumi Browser is a small, focused project. The rules are short on purpose.

## The Rules

1. **Be cool to each other.** Treat other contributors, users, and
   maintainers with basic decency. Disagreement is fine; contempt is not.
2. **No silly nonsense.** Keep discussion on-topic. This is not the place
   for trolling, flaming, recruiting for unrelated causes, drive-by
   politics, or off-topic drama.
3. **Keep it professional.** Code review is about the code. Bug reports
   are about the bug. If a thread starts to drift into personal attacks,
   identity-based insults, or harassment, stop and reset.

That's it. If you can work with those three, you can work here.

## Where This Applies

This applies to every project space — issues, pull requests, code review
comments, commit messages, the project's chat or forum if and when one
exists, and any space where you are clearly representing or contributing
to Yumi Browser.

## Enforcement

The maintainer is the final authority on what counts as breaking the
rules above. Likely responses, in increasing order:

1. A request to stop and re-engage in good faith.
2. Edit or removal of an offending comment.
3. A temporary cool-off (no participation for a period).
4. A permanent ban from project spaces for repeated or severe violations.

Decisions will be made in good faith and explained when reasonable. They
will not always be debated.

## Reporting

If you believe someone is breaking the rules — especially in a way that
involves harassment or that you do not feel safe raising in public —
contact the maintainer privately at the address listed in
[`SECURITY.md`](SECURITY.md). Reports will be handled discreetly.

## In One Sentence

Be cool, no nonsense, stay professional, and we will get along fine.
